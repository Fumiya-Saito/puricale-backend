- **Phase 4**: 信頼基盤（データエクスポート・設定UI） ✅ 完了
- **Phase 5**: ナレッジ化と資産化・サブスク化基盤 ✅ 完了（※決済実装は未了）

## 🚀 今後のリリース計画

### Phase 6: 一般公開に向けた準備（直近の必須タスク）
現状、開発環境としての制限が残っているため、誰でも使えるようにするための改修を行います。

- [ ] **ユーザー制限の撤廃**: `ALLOWED_USERS` によるハードコーディング制限を解除し、新規ユーザーが登録できるようにする。
- [ ] **Google OAuth ホームページ審査**: 認証画面で「未確認のアプリ」警告が出ないようにするため、Google Search Console の所有権確認と OAuth 審査を完了させる。
- [ ] **設定画面の安定化**: セッション切れやトークン期限切れ時（`GET /settings`）に、再認証を促すか自動リフレッシュを行うようにエラーハンドリングを強化する。

### Phase 7: サブスクリプション決済の実装（最重要ビジネス要件）
「プレミアム機能」を提供するための決済システム（Stripe連携）を組み込みます。

- [ ] **Stripe 連携基盤の構築**: Stripe Checkout を用いた月額サブスクリプション（月額300〜500円想定）の決済フローを作成。
- [ ] **Stripe Webhook の実装**: 決済成功・キャンセルをトリガーに、Supabase `users` テーブルの `is_premium` フラグを同期する。
- [ ] **復元チケットの都度購入**: プレミアムプランとは別に、「チケット1枚買い切り」などの購入フローを実装する。
- [ ] **リマインド通知の権限チェック**: 現在全ユーザーに飛んでいる行事3日前のリマインド通知を、「プレミアム会員限定」になるよう `is_premium` フラグで制御する。

### Phase 8: ユーザー体験（UX）の継続的向上
より長く使ってもらうための利便性向上施策です。

- [ ] **チケット残数の可視化**: LIFFの設定画面（`/settings`）上で、現在の契約プランとチケット残数を確認できるようにする。
- [ ] **タグの正規化と辞書化 (`tag_corrections`)**: Geminiが抽出する「運動会」「体育祭」などの表記揺れを吸収し、検索ヒット率（再利用率）を上げるための学習ループを実装。
- [ ] **通知のカスタマイズ**: リマインド通知を「3日前」だけでなく、ユーザーが設定画面からカスタマイズできるようにする。
