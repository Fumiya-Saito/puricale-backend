- **Phase 1〜3**: 基礎機能（AI解析・カレンダー登録・LINE Bot化） ✅ 完了
- **Phase 4**: 信頼基盤（設定UI・エラーハンドリング） ✅ 完了
- **Phase 5**: ナレッジ化と資産化・サブスク化基盤 ✅ 完了（Stripe決済デモ版実装済み）

## 🚀 今後のリリース計画

### Phase 6: 一般公開に向けた準備（直近の必須タスク）
現状、開発環境としての制限が残っているため、誰でも使えるようにするための改修を行います。

- [ ] **Google OAuth ホームページ審査**: 認証画面で「未確認のアプリ」警告が出ないようにするため、Google Search Console の所有権確認と OAuth 審査を完了させる。
- [x] **設定画面の安定化**: セッション切れやトークン期限切れ時（`GET /settings`）に、再認証を促すか自動リフレッシュを行うようにエラーハンドリングを強化する。

### Phase 7: サブスクリプション決済の実装（最重要ビジネス要件）
「プレミアム機能」を提供するための決済システム（Stripe連携）を組み込みます。

- [x] **Stripe 連携基盤の構築**: Stripe Checkout を用いた月額サブスクリプション（月額300〜500円想定）の決済フローを作成。
- [x] **Stripe Webhook の実装**: 決済成功・キャンセルをトリガーに、Supabase `users` テーブルの `is_premium` フラグを同期する。
- [x] **復元チケットの都度購入**: プレミアムプランとは別に、「チケット1枚買い切り」などの購入フローを実装する。
- [x] **リマインド通知の権限チェック**: 行事3日前のリマインド通知を、「プレミアム会員限定」になるよう `is_premium` フラグで制御する。

### Phase 8: ユーザー体験（UX）の継続的向上
より長く使ってもらうための利便性向上施策です。

- [x] **チケット残数の可視化**: LIFFの設定画面（`/settings`）上で、現在の契約プランとチケット残数を確認できるようにする。
- [ ] **タグの正規化と辞書化 (`tag_corrections`)**: Geminiが抽出する「運動会」「体育祭」などの表記揺れを吸収し、検索ヒット率（再利用率）を上げるための学習ループを実装。
- [ ] **通知のカスタマイズ**: リマインド通知を「3日前」だけでなく、ユーザーが設定画面からカスタマイズできるようにする。

### Phase 9: 画像ギャラリー（マイページ）とフリーミアム戦略 🌟NEW
「とりあえず見返したい」というニーズを満たしつつ、プレミアム課金への強力な導線となるWebギャラリーを実装します。

- [x] **画像一覧画面（マイページ）の実装**: 過去に送信したプリント画像を時系列・タグ別で閲覧できるLIFFページを作成。
- [x] **1ヶ月間の無料枠（Loss Aversion トリガー）**:
  - 無料ユーザーは「過去1ヶ月以内」のプリントのみフルカラーでタップ（詳細表示）可能。
  - 1ヶ月経過した画像はグレーアウトされ「鍵マーク」がつき、タップ時にプレミアム登録（またはチケット消費）を促す。
- [x] **予定連携の可視化**: 画像の詳細画面を開くと、そのプリントからAIが抽出・登録した「カレンダー予定の一覧」が画像の下部に紐付いて表示されるようにし、AIの価値を可視化する。
